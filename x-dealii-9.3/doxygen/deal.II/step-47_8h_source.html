830  * <a name="Assemblingthelinearsystem"></a>
831  * <h4>Assembling the linear system</h4>
832  *
833 
834  *
835  * The following pieces of code are more interesting. They all relate to the
836  * assembly of the linear system. While assembling the cell-interior terms
837  * is not of great difficulty -- that works in essence like the assembly
838  * of the corresponding terms of the Laplace equation, and you have seen
839  * how this works in @ref step_4 "step-4" or @ref step_6 "step-6", for example -- the difficulty
840  * is with the penalty terms in the formulation. These require the evaluation
841  * of gradients of shape functions at interfaces of cells. At the least,
842  * one would therefore need to use two FEFaceValues objects, but if one of the
843  * two sides is adaptively refined, then one actually needs an FEFaceValues
844  * and one FESubfaceValues objects; one also needs to keep track which
845  * shape functions live where, and finally we need to ensure that every
846  * face is visited only once. All of this is a substantial overhead to the
847  * logic we really want to implement (namely the penalty terms in the
848  * bilinear form). As a consequence, we will make use of the
849  * FEInterfaceValues class -- a helper class in deal.II that allows us
850  * to abstract away the two FEFaceValues or FESubfaceValues objects and
851  * directly access what we really care about: jumps, averages, etc.
852  *
853 
854  *
855  * But this doesn't yet solve our problem of having to keep track of
856  * which faces we have already visited when we loop over all cells and
857  * all of their faces. To make this process simpler, we use the
858  * MeshWorker::mesh_loop() function that provides a simple interface
859  * for this task: Based on the ideas outlined in the WorkStream
860  * namespace documentation, MeshWorker::mesh_loop() requires three
861  * functions that do work on cells, interior faces, and boundary
862  * faces. These functions work on scratch objects for intermediate
863  * results, and then copy the result of their computations into
864  * copy data objects from where a copier function copies them into
865  * the global matrix and right hand side objects.
866  *
867 
868  *
869  * The following structures then provide the scratch and copy objects
870  * that are necessary for this approach. You may look up the WorkStream
871  * namespace as well as the
872  * @ref threads "Parallel computing with multiple processors"
873  * module for more information on how they typically work.
874  *

937  * The more interesting part is where we actually assemble the linear system.
938  * Fundamentally, this function has five parts:
939  * - The definition of the `cell_worker` lambda function, a small
940  * function that is defined within the `assemble_system()`
941  * function and that will be responsible for computing the local
942  * integrals on an individual cell. It will work on a copy of the
943  * `ScratchData` class and put its results into the corresponding
944  * `CopyData` object.
945  * - The definition of the `face_worker` lambda function that does
946  * the integration of all terms that live on the interfaces between
947  * cells.
948  * - The definition of the `boundary_worker` function that does the
949  * same but for cell faces located on the boundary of the domain.
950  * - The definition of the `copier` function that is responsible
951  * for copying all of the data the previous three functions have
952  * put into copy objects for a single cell, into the global matrix
953  * and right hand side.
954  *
955 
956  *
957  * The fifth part is the one where we bring all of this together.
958  *
959 
960  *
961  * Let us go through each of these pieces necessary for the assembly
962  * in turns.
963  *
964  * @code

971  *
972  * The first piece is the `cell_worker` that does the assembly
973  * on the cell interiors. It is a (lambda) function that takes
974  * a cell (input), a scratch object, and a copy object (output)
975  * as arguments. It looks like the assembly functions of many
976  * other of the tutorial programs, or at least the body of the
977  * loop over all cells.
978  *
979 
980  *
981  * The terms we integrate here are the cell contribution
982  * @f{align*}{
983  * A^K_{ij} = \int_K \nabla^2\varphi_i(x) : \nabla^2\varphi_j(x) dx
984  * @f}
985  * to the global matrix, and
986  * @f{align*}{
987  * f^K_i = \int_K \varphi_i(x) f(x) dx
988  * @f}
989  * to the right hand side vector.
990  *
991 
992  *
993  * We use the same technique as used in the assembly of @ref step_22 "step-22"
994  * to accelerate the function: Instead of calling
995  * `fe_values.shape_hessian(i, qpoint)` in the innermost loop,
996  * we create a variable `hessian_i` that evaluates this
997  * value once in the loop over `i` and re-use the so-evaluated
998  * value in the loop over `j`. For symmetry, we do the same with a
999  * variable `hessian_j`, although it is indeed only used once and
1000  * we could have left the call to `fe_values.shape_hessian(j,qpoint)`
1001  * in the instruction that computes the scalar product between
1002  * the two terms.
1003  *
1004  * @code

1052  * The next building block is the one that assembles penalty terms on each
1053  * of the interior faces of the mesh. As described in the documentation of
1054  * MeshWorker::mesh_loop(), this function receives arguments that denote
1055  * a cell and its neighboring cell, as well as (for each of the two
1056  * cells) the face (and potentially sub-face) we have to integrate
1057  * over. Again, we also get a scratch object, and a copy object
1058  * for putting the results in.
1059  *
1060 
1061  *
1062  * The function has three parts itself. At the top, we initialize
1063  * the FEInterfaceValues object and create a new `CopyData::FaceData`
1064  * object to store our input in. This gets pushed to the end of the
1065  * `copy_data.face_data` variable. We need to do this because
1066  * the number of faces (or subfaces) over which we integrate for a
1067  * given cell differs from cell to cell, and the sizes of these
1068  * matrices also differ, depending on what degrees of freedom
1069  * are adjacent to the face or subface. As discussed in the documentation
1070  * of MeshWorker::mesh_loop(), the copy object is reset every time a new
1071  * cell is visited, so that what we push to the end of
1072  * `copy_data.face_data()` is really all that the later `copier` function
1073  * gets to see when it copies the contributions of each cell to the global
1074  * matrix and right hand side objects.

1100  *
1101  * The second part deals with determining what the penalty
1102  * parameter should be. By looking at the units of the various
1103  * terms in the bilinear form, it is clear that the penalty has
1104  * to have the form @f$\frac{\gamma}{h_K}@f$ (i.e., one over length
1105  * scale), but it is not a priori obvious how one should choose
1106  * the dimension-less number @f$\gamma@f$. From the discontinuous
1107  * Galerkin theory for the Laplace equation, one might
1108  * conjecture that the right choice is @f$\gamma=p(p+1)@f$ is the
1109  * right choice, where @f$p@f$ is the polynomial degree of the
1110  * finite element used. We will discuss this choice in a bit
1111  * more detail in the results section of this program.
1112  *
1113 
1114  *
1115  * In the formula above, @f$h_K@f$ is the size of cell @f$K@f$. But this
1116  * is not quite so straightforward either: If one uses highly
1117  * stretched cells, then a more involved theory says that @f$h@f$
1118  * should be replaced by the diameter of cell @f$K@f$ normal to the
1119  * direction of the edge in question. It turns out that there
1120  * is a function in deal.II for that. Secondly, @f$h_K@f$ may be
1121  * different when viewed from the two different sides of a face.
1122  *
1123 
1124  *
1125  * To stay on the safe side, we take the maximum of the two values.
1126  * We will note that it is possible that this computation has to be
1127  * further adjusted if one were to use hanging nodes resulting from
1128  * adaptive mesh refinement.
1129  *
1130  * @code
1131  * const unsigned int p = fe.degree;
1132  * const double gamma_over_h =
1133  * std::max((1.0 * p * (p + 1) /
1134  * cell->extent_in_direction(
1135  * GeometryInfo<dim>::unit_normal_direction[f])),
1136  * (1.0 * p * (p + 1) /
1137  * ncell->extent_in_direction(
1138  * GeometryInfo<dim>::unit_normal_direction[nf])));
1139  *
1140  * @endcode
1141  *
1142  * Finally, and as usual, we loop over the quadrature points and
1143  * indices `i` and `j` to add up the contributions of this face
1144  * or sub-face. These are then stored in the
1145  * `copy_data.face_data` object created above. As for the cell
1146  * worker, we pull the evaluation of averages and jumps out of
1147  * the loops if possible, introducing local variables that store
1148  * these results. The assembly then only needs to use these
1149  * local variables in the innermost loop. Regarding the concrete
1150  * formula this code implements, recall that the interface terms
1151  * of the bilinear form were as follows:
1152  * @f{align*}{
1153  * -\sum_{e \in \mathbb{F}} \int_{e}
1154  * \jump{ \frac{\partial v_h}{\partial \mathbf n}}
1155  * \average{\frac{\partial^2 u_h}{\partial \mathbf n^2}} \ ds
1156  * -\sum_{e \in \mathbb{F}} \int_{e}
1157  * \average{\frac{\partial^2 v_h}{\partial \mathbf n^2}}
1158  * \jump{\frac{\partial u_h}{\partial \mathbf n}} \ ds
1159  * + \sum_{e \in \mathbb{F}}
1160  * \frac{\gamma}{h_e}
1161  * \int_e
1162  * \jump{\frac{\partial v_h}{\partial \mathbf n}}
1163  * \jump{\frac{\partial u_h}{\partial \mathbf n}} \ ds.
1164  * @f}

1237  * fe_interface_values.get_normal_vectors();
1238  *
1239  *
1240  * const ExactSolution::Solution<dim> exact_solution;
1241  * std::vector<Tensor<1, dim>> exact_gradients(q_points.size());
1242  * exact_solution.gradient_list(q_points, exact_gradients);
1243  *
1244  *
1245  * @endcode
1246  *
1247  * Positively, because we now only deal with one cell adjacent to the
1248  * face (as we are on the boundary), the computation of the penalty
1249  * factor @f$\gamma@f$ is substantially simpler:
1250  *
1251  * @code
1252  * const unsigned int p = fe.degree;
1253  * const double gamma_over_h =
1254  * (1.0 * p * (p + 1) /
1255  * cell->extent_in_direction(
1256  * GeometryInfo<dim>::unit_normal_direction[face_no]));
1257  *
1258  * @endcode
1259  *
1260  * The third piece is the assembly of terms. This is now
1261  * slightly more involved since these contains both terms for
1262  * the matrix and for the right hand side. The former is exactly
1263  * the same as for the interior faces stated above if one just
1264  * defines the jump and average appropriately (which is what the
1265  * FEInterfaceValues class does). The latter requires us to
1266  * evaluate the boundary conditions @f$j(\mathbf x)@f$, which in the
1267  * current case (where we know the exact solution) we compute
1268  * from @f$j(\mathbf x) = \frac{\partial u(\mathbf x)}{\partial
1269  * {\mathbf n}}@f$. The term to be added to the right hand side
1270  * vector is then
1271  * @f$\frac{\gamma}{h_e}\int_e
1272  * \jump{\frac{\partial v_h}{\partial \mathbf n}} j \ ds@f$.
1273  *
1274  * @code
1275  * for (unsigned int qpoint = 0; qpoint < q_points.size(); ++qpoint)
1276  * {
1277  * const auto &n = normals[qpoint];
1278  *
1279  * for (unsigned int i = 0; i < n_dofs; ++i)
1280  * {
1281  * const double av_hessian_i_dot_n_dot_n =
1282  * (fe_interface_values.average_hessian(i, qpoint) * n * n);
1283  * const double jump_grad_i_dot_n =
1284  * (fe_interface_values.jump_gradient(i, qpoint) * n);
1285  *
1286  * for (unsigned int j = 0; j < n_dofs; ++j)
1287  * {
1288  * const double av_hessian_j_dot_n_dot_n =
1289  * (fe_interface_values.average_hessian(j, qpoint) * n * n);
1290  * const double jump_grad_j_dot_n =
1291  * (fe_interface_values.jump_gradient(j, qpoint) * n);
1292  *
1293  * copy_data_face.cell_matrix(i, j) +=
1294  * (-av_hessian_i_dot_n_dot_n // - {grad^2 v n n}
1295  * * jump_grad_j_dot_n // [grad u n]
1296  *
1297  * - av_hessian_j_dot_n_dot_n // - {grad^2 u n n}
1298  * * jump_grad_i_dot_n // [grad v n]
1299  *
1300  * + gamma_over_h // gamma/h
1301  * * jump_grad_i_dot_n // [grad v n]
1302  * * jump_grad_j_dot_n // [grad u n]
1303  * ) *
1304  * JxW[qpoint]; // dx
1305  * }
1306  *
1307  * copy_data.cell_rhs(i) +=
1308  * (-av_hessian_i_dot_n_dot_n * // - {grad^2 v n n }
1309  * (exact_gradients[qpoint] * n) // (grad u_exact . n)
1310  * + // +
1311  * gamma_over_h // gamma/h
1312  * * jump_grad_i_dot_n // [grad v n]
1313  * * (exact_gradients[qpoint] * n) // (grad u_exact . n)
1314  * ) *
1315  * JxW[qpoint]; // dx
1316  * }
1317  * }
1318  * };
1319  *
1320  * @endcode
1321  *
1322  * Part 4 is a small function that copies the data produced by the
1323  * cell, interior, and boundary face assemblers above into the
1324  * global matrix and right hand side vector. There really is not
1325  * very much to do here: We distribute the cell matrix and right
1326  * hand side contributions as we have done in almost all of the
1327  * other tutorial programs using the constraints objects. We then
1328  * also have to do the same for the face matrix contributions
1329  * that have gained content for the faces (interior and boundary)
1330  * and that the `face_worker` and `boundary_worker` have added
1331  * to the `copy_data.face_data` array.
1332  *
1333  * @code
1334  * auto copier = [&](const CopyData &copy_data) {
1335  * constraints.distribute_local_to_global(copy_data.cell_matrix,
1336  * copy_data.cell_rhs,
1337  * copy_data.local_dof_indices,
1338  * system_matrix,
1339  * system_rhs);
1340  *
1341  * for (auto &cdf : copy_data.face_data)
1342  * {
1343  * constraints.distribute_local_to_global(cdf.cell_matrix,
1344  * cdf.joint_dof_indices,
1345  * system_matrix);
1346  * }
1347  * };
1348  *
1349  *
1350  * @endcode
1351  *
1352  * Having set all of this up, what remains is to just create a scratch
1353  * and copy data object and call the MeshWorker::mesh_loop() function
1354  * that then goes over all cells and faces, calls the respective workers
1355  * on them, and then the copier function that puts things into the
1356  * global matrix and right hand side. As an additional benefit,
1357  * MeshWorker::mesh_loop() does all of this in parallel, using
1358  * as many processor cores as your machine happens to have.
1359  *
1360  * @code
1361  * const unsigned int n_gauss_points = dof_handler.get_fe().degree + 1;
1362  * ScratchData<dim> scratch_data(mapping,
1363  * fe,
1364  * n_gauss_points,
1365  * update_values | update_gradients |

1373  * dof_handler.end(),
1374  * cell_worker,

1383  * }

1386  *

1392  *
1393 
1394  *
1395  * The show is essentially over at this point: The remaining functions are
1396  * not overly interesting or novel. The first one simply uses a direct
1397  * solver to solve the linear system (see also @ref step_29 "step-29"):
1398  *
1399  * @code
1400  * template <int dim>

1402  * {

1404  *

1407  * A_direct.vmult(solution, system_rhs);
1408  *

1410  * }

1412  *
1413  *
1414  * @endcode
1415  *
1416  * The next function evaluates the error between the computed solution
1417  * and the exact solution (which is known here because we have chosen
1418  * the right hand side and boundary values in a way so that we know
1419  * the corresponding solution). In the first two code blocks below,
1420  * we compute the error in the @f$L_2@f$ norm and the @f$H^1@f$ semi-norm.
1421  *
1422  * @code
1423  * template <int dim>

1425  * {

1429  * dof_handler,
1430  * solution,
1431  * ExactSolution::Solution<dim>(),
1432  * norm_per_cell,
1433  * QGauss<dim>(fe.degree + 2),
1434  * VectorTools::L2_norm);

1436  * VectorTools::compute_global_error(triangulation,
1437  * norm_per_cell,
1438  * VectorTools::L2_norm);
1439  * std::cout << " Error in the L2 norm : " << error_norm
1440  * << std::endl;
1441  * }
1442  *
1443  * {
1444  * Vector<float> norm_per_cell(triangulation.n_active_cells());
1445  * VectorTools::integrate_difference(mapping,
1446  * dof_handler,
1447  * solution,
1448  * ExactSolution::Solution<dim>(),
1449  * norm_per_cell,
1450  * QGauss<dim>(fe.degree + 2),
1451  * VectorTools::H1_seminorm);
1452  * const double error_norm =
1453  * VectorTools::compute_global_error(triangulation,
1454  * norm_per_cell,
1455  * VectorTools::H1_seminorm);

1457  * << std::endl;
1458  * }

1460  * @endcode
1461  *
1462  * Now also compute an approximation to the @f$H^2@f$ seminorm error. The actual
1463  * @f$H^2@f$ seminorm would require us to integrate second derivatives of the
1464  * solution @f$u_h@f$, but given the Lagrange shape functions we use, @f$u_h@f$ of
1465  * course has kinks at the interfaces between cells, and consequently second
1466  * derivatives are singular at interfaces. As a consequence, we really only
1467  * integrate over the interior of cells and ignore the interface
1468  * contributions. This is *not* an equivalent norm to the energy norm for
1469  * the problem, but still gives us an idea of how fast the error converges.
1470  *
1471 
1472  *
1473  * We note that one could address this issue by defining a norm that
1474  * is equivalent to the energy norm. This would involve adding up not
1475  * only the integrals over cell interiors as we do below, but also adding
1476  * penalty terms for the jump of the derivative of @f$u_h@f$ across interfaces,
1477  * with an appropriate scaling of the two kinds of terms. We will leave
1478  * this for later work.

1480  * @code
1481  * {

1488  * quadrature_formula,

1494  *
1495  * std::vector<SymmetricTensor<2, dim>> exact_hessians(n_q_points);
1496  * std::vector<Tensor<2, dim>> hessians(n_q_points);
1497  * for (auto &cell : dof_handler.active_cell_iterators())
1498  * {
1499  * fe_values.reinit(cell);
1500  * fe_values[scalar].get_function_hessians(solution, hessians);
1501  * exact_solution.hessian_list(fe_values.get_quadrature_points(),
1502  * exact_hessians);
1503  *
1504  * double local_error = 0;
1505  * for (unsigned int q_point = 0; q_point < n_q_points; ++q_point)
1506  * {
1507  * local_error +=
1508  * ((exact_hessians[q_point] - hessians[q_point]).norm_square() *
1509  * fe_values.JxW(q_point));
1510  * }
1511  * error_per_cell[cell->active_cell_index()] = std::sqrt(local_error);
1512  * }
1513  *
1514  * const double error_norm = error_per_cell.l2_norm();
1515  * std::cout << " Error in the broken H2 seminorm: " << error_norm
1516  * << std::endl;
1517  * }
1518  * }
1519  *
1520  *
1521  *
1522  * @endcode

1646 We run the program with right hand side and boundary values as
1647 discussed in the introduction. These will produce the
1648 solution @f$u = \sin(\pi x) \sin(\pi y)@f$ on the domain @f$\Omega = (0,1)^2@f$.
1649 We test this setup using @f$Q_2@f$, @f$Q_3@f$, and @f$Q_4@f$ elements, which one can
1650 change via the `fe_degree` variable in the `main()` function. With mesh
1651 refinement, the @f$L_2@f$ convergence rates, @f$H^1@f$-seminorm rate,

1657 From the literature, it is not immediately clear what

1659 @cite Brenner2009 state that it needs to be larger than one, and
1660 choose @f$\gamma=5@f$. The FEniCS/Dolphin tutorial chooses it as

1667 @f$p(p+1)@f$ would be reasonable,

1669 the one one would expect to work by comparing
1670 to the discontinuous Galerkin formulations for the Laplace equation
1671 (see, for example, the discussions in @ref step_39 "step-39" and @ref step_74 "step-74"),
1672 and it will turn out to also work here.

1674 below; changing @f$\gamma@f$ is easy in the two `face_worker` and

1867 <a name="Derivationforthesimplysupportedplates"></a> <h4> Derivation for the simply supported plates </h4>
1868 
1869 
1870  Similar to the "clamped" boundary condition addressed in the implementation,
1871  we will derive the @f$C^0@f$ IP finite element scheme for simply supported plates:
1872  @f{align*}{
1873  \Delta^2 u(\mathbf x) &= f(\mathbf x)
1874  \qquad \qquad &&\forall \mathbf x \in \Omega,
1875  u(\mathbf x) &= g(\mathbf x) \qquad \qquad
1876  &&\forall \mathbf x \in \partial\Omega, \\
1877  \Delta u(\mathbf x) &= h(\mathbf x) \qquad \qquad
1878  &&\forall \mathbf x \in \partial\Omega.
1879  @f}
1880  We multiply the biharmonic equation by the test function @f$v_h@f$ and integrate over @f$ K @f$ and get:
1881  @f{align*}{
1882  \int_K v_h (\Delta^2 u_h)
1883  &= \int_K (D^2 v_h) : (D^2 u_h)
1884  + \int_{\partial K} v_h \frac{\partial (\Delta u_h)}{\partial \mathbf{n}}
1885  -\int_{\partial K} (\nabla v_h) \cdot (\frac{\partial \nabla u_h}{\partial \mathbf{n}}).
1886  @f}
1887 
1888  Summing up over all cells @f$K \in \mathbb{T}@f$,since normal directions of @f$\Delta u_h@f$ are pointing at
1889  opposite directions on each interior edge shared by two cells and @f$v_h = 0@f$ on @f$\partial \Omega@f$,
1890  @f{align*}{
1891  \sum_{K \in \mathbb{T}} \int_{\partial K} v_h \frac{\partial (\Delta u_h)}{\partial \mathbf{n}} = 0,
1892  @f}
1893  and by the definition of jump over cell interfaces,
1894  @f{align*}{
1895  -\sum_{K \in \mathbb{T}} \int_{\partial K} (\nabla v_h) \cdot (\frac{\partial \nabla u_h}{\partial \mathbf{n}}) = -\sum_{e \in \mathbb{F}} \int_{e} \jump{\frac{\partial v_h}{\partial \mathbf{n}}} (\frac{\partial^2 u_h}{\partial \mathbf{n^2}}).
1896  @f}
1897  We separate interior faces and boundary faces of the domain,
1898  @f{align*}{
1899  -\sum_{K \in \mathbb{T}} \int_{\partial K} (\nabla v_h) \cdot (\frac{\partial \nabla u_h}{\partial \mathbf{n}}) = -\sum_{e \in \mathbb{F}^i} \int_{e} \jump{\frac{\partial v_h}{\partial \mathbf{n}}} (\frac{\partial^2 u_h}{\partial \mathbf{n^2}})
1900  - \sum_{e \in \partial \Omega} \int_{e} \jump{\frac{\partial v_h}{\partial \mathbf{n}}} h,
1901  @f}
1902  where @f$\mathbb{F}^i@f$ is the set of interior faces.
1903  This leads us to
1904  @f{align*}{
1905  \sum_{K \in \mathbb{T}} \int_K (D^2 v_h) : (D^2 u_h) \ dx - \sum_{e \in \mathbb{F}^i} \int_{e} \jump{\frac{\partial v_h}{\partial \mathbf{n}}} (\frac{\partial^2 u_h}{\partial \mathbf{n^2}}) \ ds
1906  = \sum_{K \in \mathbb{T}}\int_{K} v_h f \ dx + \sum_{e\subset\partial\Omega} \int_{e} \jump{\frac{\partial v_h}{\partial \mathbf{n}}} h \ ds.
1907  @f}
1908 
1909  In order to symmetrize and stabilize the discrete problem,
1910  we add symmetrization and stabilization term.
1911  We finally get the @f$C^0@f$ IP finite element scheme for the biharmonic equation:
1912  find @f$u_h@f$ such that @f$u_h =g@f$ on @f$\partial \Omega@f$ and
1913  @f{align*}{
1914  \mathcal{A}(v_h,u_h)&=\mathcal{F}(v_h) \quad \text{holds for all test functions } v_h,
1915  @f}
1916  where
1917  @f{align*}{
1918  \mathcal{A}(v_h,u_h):=&\sum_{K \in \mathbb{T}}\int_K D^2v_h:D^2u_h \ dx
1919  \\
1920  &
1921  -\sum_{e \in \mathbb{F}^i} \int_{e}
1922  \jump{\frac{\partial v_h}{\partial \mathbf n}}
1923  \average{\frac{\partial^2 u_h}{\partial \mathbf n^2}} \ ds
1924  -\sum_{e \in \mathbb{F}^i} \int_{e}
1925  \average{\frac{\partial^2 v_h}{\partial \mathbf n^2}}
1926  \jump{\frac{\partial u_h}{\partial \mathbf n}} \ ds
1927  \\
1928  &+ \sum_{e \in \mathbb{F}^i}
1929  \frac{\gamma}{h_e}
1930  \int_e
1931  \jump{\frac{\partial v_h}{\partial \mathbf n}}
1932  \jump{\frac{\partial u_h}{\partial \mathbf n}} \ ds,
1933  @f}
1934  and
1935  @f{align*}{
1936  \mathcal{F}(v_h)&:=\sum_{K \in \mathbb{T}}\int_{K} v_h f \ dx
1937  +
1938  \sum_{e\subset\partial\Omega}
1939  \int_e \jump{\frac{\partial v_h}{\partial \mathbf n}} h \ ds.
1940  @f}
1941  The implementation of this boundary case is similar to the "clamped" version
1942  except that `boundary_worker` is no longer needed for system assembling
